Conversational RAG over COVID-19 Doctor Interviews — Pipeline

Current config: MiniLM embeddings · hybrid (dense + BM25 → RRF) · extractive generation · math judge

flowchart TD
  subgraph INGEST["① Ingestion & Processing"]
    A[PDF / DOCX transcripts
20 interviews] --> B[Clean & normalize text]
    B --> C["Q&A-aware chunking
split on 'Interviewer:'
question + answer kept together"]
    C --> D["Extract metadata + topics
doctor · country · specialty · topic"]
  end
  subgraph INDEX["② Hybrid Index (persisted to disk)"]
    D --> E["Dense embeddings
MiniLM · 384-dim"]
    D --> F["BM25 lexical index"]
    E --> G[(Hybrid Store
dense.npy + chunks.json + BM25)]
    F --> G
  end
  Q([User question]) --> R
  subgraph RETRIEVE["③ Hybrid Retrieval Engine"]
    R["Score every chunk:
dense cosine + BM25"] --> S["Reciprocal Rank Fusion (RRF)"]
    S --> X[Top-k chunks + scores]
  end
  G -.loads.-> R
  X --> GATE{"Fallback gate:
dense best ≥ threshold?
(opt-in hybrid: OR BM25 best ≥ threshold)"}
  GATE -->|no| FB["⚠️ Fallback answer + reason
(anti-hallucination)"]
  GATE -->|yes| GEN
  subgraph RESPOND["④ Response Engine"]
    GEN["Generate answer
extractive (default) · or LLM"] --> CITE["Attach line-level citations
quoted line + literal source line no.
+ view-in-transcript highlight"]
  end
  CITE --> ANS([Grounded answer + citations])
  FB --> ANS
  subgraph EVAL["⑤ Evaluation Pipeline (self-scoring) — 4 MECE sections"]
    GD[(Golden dataset
18 Q&A + expected chunks/citations)] --> EV["Run engine on each question"]
    EV --> JUDGE["Judge: math (default) · or LLM-as-judge"]
    JUDGE --> M1["① Context Relevance (hero)
+ Context Recall · Context Precision"]
    JUDGE --> M2["② Groundedness (hero)
+ Citation Grounding"]
    JUDGE --> M3["③ Answer Relevance (hero)
+ Answer Correctness"]
    JUDGE --> M4["④ Ops
Fallback Correctness · Latency"]
    M1 & M2 & M3 --> TRIAD["RAG Triad = mean of the 3 heroes"]
  end
  ANS -.evaluated by.-> EV
